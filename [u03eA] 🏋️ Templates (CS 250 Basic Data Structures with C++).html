Introduction

Templates are an important part of writing generalized code that can be used in many contexts. For example, you don't want to have to write a list class that only works for one data type - why not cover all data types at once? With templates!

Goals:

  • Practice creating templated functions
  • Practice creating templates classes

Templated function

First, we're going to create a templated function. We'll create a function that takes a parameter and display it in a column. It will take an optional second parameter to define the width of the column.

This function will go in your functions.hpp file.

Don't forget the file guards!

Remember that all header files (.h or .hpp files) need to have file guards to prevent duplicate code imports. The file guards look like this:

#ifndef _YOUR_LABEL
#define _YOUR_LABEL

// Code goes here

#endif

The label can be anything so long as each header file has its own unique label. Usually, I name it something related to the filename, like _FUNCTIONS_HPP.

To create our templated function, we start with specifying that it's a template and an alias for the datatype (usually just "T").

Then, in the parameter list, for any parameters that we want to allow any data type, we use "T". For parameters that always need to be the same data type, we can just write it as normal.

Within this function, we display the col item passed in, using left and setw from iomanip in order to create a column of some width (the colWidth passed in).

This all goes in the header file functions.hpp:

template <typename T>
void DisplayColumn( T col, int colWidth = 20 )
{
    cout << left << setw( colWidth ) << col;
}

Default parameters:

The second parameter here has a default value assigned to it: int colWidth = 20. What this means is that, if somebody calls the function like this: 

DisplayColumn( myVariable );
The code will still run! The parameter colWidth will have a value of 20 if nothing is specified.

Of course, you could also call the function and explicitly set that parameter's value via an argument: 
DisplayColumn( myVariable, 10 );
Both of these calls will work!

Calling the function from main()

In your main() fucntion, no special symbols or keywords are required to call this templated function. We can pass in any data type as the first argument, and we can either leave off the second

argument or supply an integer to specify how wide the column should be. For example: 

cout << "Column width 20:" << endl;
DisplayColumn( "NAME" );
DisplayColumn( "PRICE" );

cout << endl << endl;
cout << "Column width 10:" << endl;
DisplayColumn( "NAME", 10 );
DisplayColumn( "PRICE", 10 );

Column width 20:
NAME                PRICE               

Column width 10:
NAME      PRICE

Templated class

Templated classes can be useful for creating storage for any data type. For this example, we will create a class that wraps a basic fixed-length array, but with a templated data type. This function will also need a function to add new elements to the array and to retrieve a value from the array at some index.

When creating a templated class, the class declaration and class' method definitions all need to go into the header file (whereas normally, we'd put declarations in the header and definitions in the source). This is because C++ is old and weird. There are ways around this, but I think it's clearer to just put it all in the header.

Within classes.hpp, make sure to write the file guard. Then, write out the class declaration. The only thing that will differ from a normal class declaration is:

  • You will put template <typename T> before the class Storage line
  • The array that stores the data (of any data type) should have the data type "T".
  • Any parameters that take in any data type, and any return types that return any data type, should use "T". In this case, the Add function takes in a new element for the array, and the Get function returns an item from the array.

Can I see how the class should look?
#ifndef _CLASSES_HPP
#define _CLASSES_HPP

template <typename T>
class Storage
{
    public:
    Storage()
        : ARRAY_SIZE( 20 ) // initialize the const int
    {
      // add code
    }

    void Add( T newItem )
    {
      // add code
    }

    T Get( int index ) const
    {
      return m_arr[0]; // replace me
    }

    int Size() const
    {
      return -1; // replace me
    }

    private:
    const int ARRAY_SIZE;
    int m_itemCount;
    T m_arr[20];
};

#endif
Storage
- m_arr[20] : T
- m_itemCount : int
- ARRAY_SIZE : const int
+ Storage()
+ Add( newItem : T ) : void
+ Get( index : int ) const : T
+ Size() const : int


Why do you have m_ as prefixes?

There are many different coding styles, but in some style guides they use _ or m_ to prefix any private member variables that belong to a class.

I like to do this because then with a Setter, I can write this: 

void SetItemCount( int itemCount )
{
  m_itemCount = itemCount;
}
Instead of this: 
void SetItemCount( int itemCount )
{
  this->itemCount = itemCount;
}

Storage() constructor

The default constructor needs to initialize the ARRAY_SIZE named constant (which has to be done via a member initializer list) and the m_itemCount variable. Since the m_arr array is set to size 20, the ARRAY_SIZE should also be 20. The m_itemCount should start at 0, since no items will be stored in the array to begin with.

How do I initialize the named constant with a member initializer list??

The constructor will look like this (within the class declaration):

Storage()
  : ARRAY_SIZE( 20 )
{
  m_itemCount = 0;
}

void Add( T newItem )

This function should add a new item to the array, the newItem passed in as a parameter. You should have an error check first, however, so that if the array is full, then it should display an error and return early.

If the array is not full, then you can add the new item at the index stored by m_itemCount. Make sure to increment the m_itemCount by 1 afterwards.

How do I know if the array is full?

The array is full if the m_itemCount, the amount of items we have stored in the array, is equal to the ARRAY_SIZE, which is the amount of elements the array can store. 

if ( m_itemCount == ARRAY_SIZE ) { /* It's full! */ }
What index do I add the new item at?

When the array is empty, m_itemCount will be 0. Adding the first item also goes to the index position 0.

When there is one item in the array, m_itemCount will be 1. Adding the second item will go to the position 1.

Therefore, m_itemCount has two purposes: It stores how many items we currently have in the array and the same number is also the index where the next new item will go. So: 

m_arr[ m_itemCount ] = newItem;
How do I increment m_itemCount?

You can increment with the ++ operator or += operator or write it out as a full statement. 

m_itemCount++;
m_itemCount += 1;
m_itemCount = m_itemCount + 1;

T Get( int index ) const

This function will return the element of the m_arr array at the position index, which is passed in as a parameter. We are not going to bother with any error checks for this function for now. Just return the correct element.

How do I know what to return?

Just return the element of m_arr at the index specified:

return m_arr[index];
Why aren't we doing an error check here?

Since this function has a return type that isn't void and we don't know the data type (it's "T"), this means we have to return a value. If the user enters an index that is invalid, we can't just return a default value at this point. The proper way to deal with an error here is by throwing an exception, but we will cover that later.

int Size() const

This function should return the value of m_itemCount.

Back in main()...

Back in main(), we're going to create a couple of Storage objects to store product names (strings) and product prices (floats).

They can be declared like this:

Storage<string> productNames;
Storage<float> productPrices;

Now, whatever we put within the < and > is what gets "filled in" to that "T" data type we were using in the class - the member array, the parameter for Add, and the return type for Get.

We can add items to each object via the Add function: 

productNames.Add( "Sketchbook" );
productPrices.Add( 5.59 );

Add at least a couple more product names and a couple more product prices.

Finally, let's display a list of products' names and prices. We can use our templated DisplayColumn function to display a header for our table: 

DisplayColumn( "NAME" );
DisplayColumn( "PRICE" );
cout << endl;

NAME                PRICE

Next, we can write a for loop to iterate from i = 0 to the size of one of the arrays (productNames.Size()), going up by 1 each time. Within the loop, use the DisplayColumn function to display each name and price: 

for ( int i = 0; i < productNames.Size(); i++ )
{
  DisplayColumn( productNames.Get( i ) );
  DisplayColumn( productPrices.Get( i ) );
  cout << endl;
}

Art pens            12.99               
Colored pencils     5.99

Your own templated function

Now you'll use what we've covered so far to create a templated function on your own. We're actually going to overload some functions as well to get all the outputs we want.

These functions are going to take an ofstream (output file stream) as well as any additional data and is responsible for outputting it to a text file, organized in columns.

There will be three versions of the Output function, but only one will be templated. Let's start with the first one...

void Output( ofstream& output )

The first version of this function takes in an output file stream and is only going to write a new line to the file (with endl), like this: 

void Output( ofstream& output )
{
    output << endl;
}

void Output( ofstream& output, string label )

The next version of the Output function will take an additional parameter - a string that will be output to the text file. The label will be written in a column of width 20, using the iomanip keywords: 

void Output( ofstream& output, string label )
{
    output << left << setw( 20 ) << label << endl;
}

Overloaded functions: Two or more functions can have the same name as long as their function signatures are unique. The function signature is made up of the function name and the parameter list.

In this case, one function has one parameter and the other has two, so the compiler can easily tell these apart, which means we can reuse the same function name here.

void Output( ofstream& output, string label, T data )

Finally, you'll implement a templated function version of Output, which takes in both a label and a piece of data - of any data type. In the output, it should display the label first and then the data, and both will have columns of width 20 (use setw( 20 )).

Finish implementing the function and then we'll use it in main().

main()

In main, after the previous examples, we will use the Output function to display product names and prices to a text file.

First, make sure to create an ofstream object and open a file, such as "output.txt".
